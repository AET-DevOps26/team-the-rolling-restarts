# 01 — Team Organisation & Collaboration

## Team

- Teams consist of **three students**.
- Each student must **own one primary subsystem** — typically **client, server, or
  GenAI**.
- Ownership does **not** mean isolated work. Teams must collaborate across subsystem
  boundaries, **especially for integration, deployment, and debugging**. A project
  where each student only touches their own part and skips system integration does
  **not** meet the intended DevOps model.

## Registration (required per student)

To make contribution tracking possible, each student must provide:

- **GitHub username**
- **TUMonline login**
- **Matriculation number**

## Contribution tracking

Work must be **transparent and traceable**. Contributions must be visible through:

- GitHub **commits**,
- **pull request authorship**,
- **code review participation**,
- involvement in **infrastructure tasks** (CI/CD configuration, deployment).

## Communication

- Only **official course channels in Artemis** count for evaluation.
- Tutor feedback, planning, questions, and issue reporting must be visible in the
  dedicated Artemis team channels.
- No other communication channel will be taken into account.
